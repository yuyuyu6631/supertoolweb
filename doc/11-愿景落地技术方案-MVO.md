# 愿景落地技术方案（MVO）

> 本文档是未来规划文档，不代表当前代码已经全部具备这些能力。  
> 当前真实实现基线请优先阅读 `doc/08`、`doc/10`、`doc/12`、`doc/13`、`doc/14`。

## 1. 文档目标

本文档用于回答两个问题：

1. 当前基线已经具备哪些能力，可以作为后续扩展起点
2. 在不推翻现有代码的前提下，未来还需要补哪些业务与技术能力

## 2. 当前基线已经具备什么

当前代码已经落地的基础能力：

- `apps/web` 已是唯一公开前端
- `apps/api` 已是唯一后端入口
- 前端已迁移到 `Next.js App Router`
- 后端已迁移到真实数据库模型与迁移体系
- 目录、榜单、场景、推荐接口已可用
- 基础账号认证已可用
- 抓取任务入口和定时调度已具备骨架

这意味着，未来规划不再是“从 Vite Demo 迁移到正式主工程”，而是“在现有正式主工程基础上继续补齐运营和商业化能力”。

## 3. 当前仍缺什么

距离真正的 MVO 仍有明显缺口：

- 后台管理端未落地
- 抓取、审核、发布仍未形成完整闭环
- 内容生产和 SEO 专题体系未落地
- 商业化转化链路未落地
- 共享类型仍未完全收敛
- 认证仍是基础账号密码方案，未扩展权限和后台角色体系

## 4. 推荐的未来目标

### 4.1 内容与运营

- 建立工具审核与发布后台
- 建立榜单、场景、专题内容管理能力
- 建立抓取快照到正式发布的审核流

### 4.2 推荐与搜索

- 在当前推荐接口上继续增强召回与排序
- 把推荐链路从“可用”提升到“可解释、可追踪、可优化”
- 增加日志、点击与效果分析

### 4.3 认证与权限

- 从基础用户认证扩展到后台角色体系
- 区分访客、普通用户、编辑、管理员
- 为后台能力补足权限边界

### 4.4 商业化与增长

- 建立福利、合作链接和线索收集能力
- 建立转化归因与数据分析
- 为目录、榜单、专题页补足 SEO 输出

## 5. 推荐的演进原则

- 不推翻当前 `Next.js + FastAPI + MySQL` 基线
- 不再把历史 `Vite + React Router + mock API` 视为主线方案
- 先补运营闭环，再扩充复杂商业能力
- 未来文档必须明确区分“已实现”和“规划中”

## 6. 分阶段建议

### 阶段 A：补齐运营闭环

- 后台登录与角色
- 工具内容管理
- 抓取审核发布

### 阶段 B：补齐内容分发

- 专题页
- SEO 输出
- 内容生产模版

### 阶段 C：补齐商业化

- 福利配置
- 线索收集
- 转化归因

## 7. 最终建议

当前项目已经拥有可以持续演进的正式主干。  
未来工作的重点不再是“替换主框架”，而是“围绕现有主框架补齐运营、内容、推荐、权限和商业化能力”。
