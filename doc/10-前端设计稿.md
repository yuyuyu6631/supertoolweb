# 前端落地设计说明

本文档描述的是当前代码已经落地的前端结构与设计实现，不是历史方案，也不是纯视觉概念稿。

## 1. 当前前端技术基线

- 框架：`Next.js 15 App Router`
- 运行时：`React 19`
- 样式：`Tailwind CSS 4` + `theme.css`
- 测试：`Vitest` + `Playwright`

结论：

- 当前项目是 Next.js 项目
- 当前项目不是 `Vite + React Router` 项目
- 当前页面已接入真实 API 和基础认证能力

## 2. 当前页面结构

### 2.1 首页 `/`

已落地区块：

- 顶部 Header
- Hero 搜索入口
- 预设视图卡片
- 常用分类
- 精选工具卡片
- Footer

页面特点：

- 首屏直接展示搜索和目录入口
- 首页支持跳转到工具目录并带搜索参数
- Header 显示登录状态

### 2.2 工具目录 `/tools`

已落地区块：

- 面包屑
- 搜索条
- 视图切换
- 排序切换
- 分类筛选
- 状态筛选
- 标签筛选
- 列表结果区
- 分页导航

页面特点：

- 当前是最完整的目录检索页
- 所有筛选都基于 URL 参数驱动
- 目录取数失败时有错误与超时提示

### 2.3 工具详情 `/tools/[slug]`

已落地能力：

- 工具基础信息展示
- 官网跳转
- 标签、状态、评分等信息展示

### 2.4 榜单页 `/rankings`

已落地能力：

- 多个榜单分区展示
- 榜单项复用目录工具信息

### 2.5 场景页 `/scenarios` 与 `/scenarios/[slug]`

已落地能力：

- 场景列表展示
- 场景详情说明
- 场景下的工具组合展示

### 2.6 认证页 `/auth`

已落地区块：

- 品牌说明区
- 登录/注册 Tab
- 登录表单
- 注册表单
- 社交登录占位按钮
- 已登录状态卡片

页面特点：

- 登录和注册切换在单页内完成
- 校验信息采用自然语言提示
- 已登录状态下支持退出登录

## 4. 当前视觉语言

- 浅色玻璃工作台风格
- 大圆角卡片与面板
- 大量使用半透明白色和细边框
- Header 与表单面板保持同一套玻璃语义
- 认证页延续站点主视觉，不另起一套旧风格

## 5. 当前交互状态

- Header 移动端菜单开关
- 路由切换反馈
- 目录筛选和分页
- 登录态读取中状态
- 登录/注册表单校验
- 表单提交 loading
- 登录成功后的状态切换

## 6. 当前与历史方案的差异

- 历史文档中的 `Vite`、`React Router`、浏览器 `mock API` 已不是现状
- 历史方案中的“登录注册不做”已不成立
- 当前项目已进入真实前后端协作阶段，前端设计需要围绕真实接口和登录态继续演进
