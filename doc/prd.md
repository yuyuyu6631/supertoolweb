# 星点评 - AI工具评测导航平台
## 产品需求文档（PRD）

---

## 1. 产品目标（做什么）

帮普通人 **快速找到合适的AI工具**。

现在市面上有几千种AI工具，普通人：
- 不知道有哪些AI工具
- 不知道哪个工具能解决自己的问题
- 不知道工具好不好用、贵不贵
- 懒得自己一个个对比测试

**星点评解决这个问题**：
我们帮你整理评测好了，通过榜单推荐、场景分类和AI智能推荐，30秒就能帮你找到想要的工具。

**一句话定位**：AI领域的大众点评，找AI工具就上星点评。

---

## 2. 用户是谁（谁在用）

| 用户群体 | 特征 | 核心需求 |
|---------|------|---------|
| **想尝试AI的办公族** | 职场白领，日常写文案、做PPT、处理数据 | 想提高效率，但不知道用什么AI |
| **内容创作者** | 小编、博主、自媒体、设计师 | 需要找生成文案、图片、视频的AI工具 |
| **创业者/产品经理** | 需要找AI来做竞品分析、用户调研、原型设计 | 要靠谱的工具，不踩坑 |
| **AI从业者** | 也会来看看有什么新工具出了 | 了解行业动态 |

**核心用户**：前三者，**不懂技术但想用好AI** 的办公室人群。

## 3. 使用场景

1. **小明想写文案** → 打开星点评 → 看看"文案写作"分类有哪些工具 → 看看排行榜第一名是什么 → 点进去看详细评测 → 去官网试用

2. **小丽不知道用什么做PPT** → 打开星点评 → 问AI"帮我推荐一个自动做PPT的工具" → AI给出3个推荐，说明各自优缺点 → 选一个最合适的

3. **老王做电商需要修图** → 打开星点评 → 找到"图片处理"场景 → 看看场景推荐了哪些工具组合 → 选一个免费够用的

4. **运营同学想找AI做短视频** → 搜索"短视频" → 筛选"免费"标签 → 按评分排序 → 选评分最高的几个去看

5. **朋友推荐了一个AI工具** → 来星点评搜一搜 → 看看大家评分怎么样，有没有更好的替代

## 5. 核心功能说明

### 5.1 工具目录浏览
**做什么**：让用户可以按分类浏览所有收录的AI工具，支持搜索和筛选。  
**怎么用**：
- 在搜索框输入工具名称或关键词，实时出结果
- 点击左侧分类，只看这个分类下的工具
- 勾选"免费"，只显示免费工具
- 点击"热门优先"，按热门程度排序
- 上下翻页看更多工具

### 5.2 工具详情展示
**做什么**：展示一个AI工具的完整信息，帮用户判断要不要用。  
**怎么用**：
- 顶部直接显示工具名称、评分、开发者、价格标签
- 有一个直达官网的按钮，点击直接跳转到工具官网
- 下面是详细介绍：这个工具是干什么的，有什么特色功能
- 显示标签：比如"文案写作"、"免费"、"国内可用"

### 5.3 榜单专区
**做什么**：我们编辑整理的专题排行榜，帮用户快速建立认知。  
**怎么用**：
- 榜单页显示所有专题榜单，比如"2025年最好用的5个AI写作工具"
- 点击进入榜单详情，从上到下看排名
- 每个榜单条目都有一句话点评，为什么排这个名次
- 点击工具卡片直接跳转到工具详情页

### 5.4 场景推荐
**做什么**：按实际工作生活场景推荐工具组合，不是冷冰冰的工具列表，而是解决方案。  
**怎么用**：
- 场景列表页展示所有场景：比如"HR招人"、"学生写论文"、"电商做图"
- 点击进入场景详情，先告诉你这个场景下用AI能解决什么问题
- 然后分工具推荐：哪个工具适合入门，哪个适合专业，哪个免费
- 每个推荐都说明"什么时候选它"，直接点击去看工具详情

### 5.5 AI智能推荐
**做什么**：用户用自己的话说需求，AI直接推荐最合适的工具。  
**怎么用**：
- 在首页或者推荐页输入框，用中文说你的需求，比如"帮我推荐一个能自动总结会议记录的AI工具"
- 点击"发送"按钮，等待几秒钟
- AI会给出1-3个推荐结果，每个推荐说明：推荐理由、适合什么情况、不适合什么情况
- 点击推荐卡片直接看工具详情

## 6. 页面设计说明

### 6.1 首页
| 位置 | 元素 | 交互说明 |
|-----|------|---------|
| 顶部 | 网站Logo + 导航栏 | 导航链接：首页/工具目录/榜单/场景/AI推荐 |
| 首屏 | **大搜索框** + "找AI工具"按钮 | 用户进来第一眼就可以搜索 |
| 搜索框下方 | AI推荐入口 → "说你的需求，AI帮你找" | 点击跳转到AI推荐，或者直接在这里输入 |
| 下面 | **分类导航区** | 显示主要分类卡片，比如"文案写作"、"图像生成"、"视频制作"等，点击进去看分类下所有工具 |
| 再下面 | **精选工具推荐** | 展示3-6个热门优质工具，显示封面图、名称、评分、一句话简介，点击进去看详情 |
| 底部 | 页脚 | 简单说明网站定位，联系方式 |

### 6.2 工具目录页
| 位置 | 元素 | 交互说明 |
|-----|------|---------|
| 左侧 | 筛选区 | 分类选择、标签选择、价格筛选（全部/免费/付费），勾选后立即筛选结果 |
| 右侧顶部 | 排序选择 + 结果计数 | 下拉选：热门优先/最新优先/评分从高到低 |
| 右侧主体 | **工具卡片列表** | 每个卡片显示：图标、名称、评分、价格标签、一句话简介、标签，点击卡片进入详情页 |
| 底部 | 分页 | 上一页/下一页，或者滚动加载更多 |

### 6.3 工具详情页
| 位置 | 元素 | 交互说明 |
|-----|------|---------|
| 面包屑 | 首页 > 工具目录 > 分类 > 当前工具 | 点击可以返回上级 |
| 头部 | 工具图标 + 名称 + 评分 + 一句话简介 | 评分用星星显示 |
| 信息栏 | 开发者/官网/价格分类/标签 | 价格标签清晰标注：免费/免费额度/付费 |
| 按钮区 | **「访问官网」大按钮** | 新标签页打开官网 |
| 内容区 | 详细介绍 | 详细说明这个工具能做什么，有什么特色，适合谁用 |
| 底部 | 同类工具推荐 | 显示3个同分类其他工具，方便对比 |

### 6.4 榜单页
| 位置 | 元素 | 交互说明 |
|-----|------|---------|
| 页面标题 | 榜单专区 | |
| 主体 | **榜单卡片列表** | 每个卡片显示：榜单标题、封面图、工具数量、一句话说明，点击进入榜单详情 |

### 6.5 榜单详情页
| 位置 | 元素 | 交互说明 |
|-----|------|---------|
| 头部 | 榜单标题 + 榜单介绍 | 说明这个榜单是评什么的 |
| 主体 | 排名列表 | 从第1名开始往下排，每个排名项包含：工具卡片 + 排名理由，点击工具卡片进详情 |

### 6.6 场景列表页
同榜单页结构，场景卡片替换榜单卡片。

### 6.7 场景详情页
| 位置 | 元素 | 交互说明 |
|-----|------|---------|
| 头部 | 场景名称 + 场景解决什么问题 | 比如"HR招人场景：AI帮你快速筛简历写JD" |
| 推荐区 | 分模块推荐工具 | 比如"基础必备"、"进阶好用"、"免费备选"，每个模块下推荐1-3个工具，说明推荐理由 |
| 每个工具 | 工具卡片 + 一句话什么时候选它 | 点击进详情 |
| 底部 | 场景使用小技巧 | 告诉用户在这个场景下怎么用AI最高效 |

### 6.8 AI智能推荐页
| 位置 | 元素 | 交互说明 |
|-----|------|---------|
| 顶部 | 标题提示 | "告诉我你需要什么AI工具，我来帮你推荐" |
| 输入区 | 大文本框 + "发送"按钮 | 用户输入自然语言需求 |
| 回答区 | AI推荐结果 | 每次推荐1-3个，每个推荐包含：工具名称、推荐理由、适合情况、不适合情况，点击进详情 |
| 交互 | 用户可以换个需求重新问 | 清除历史，重新推荐 |

## 7. 用户流程（一步一步怎么用）

### 流程一：浏览找工具
```
1. 用户打开网站 → 看到首页
2. 点击感兴趣的分类 → 进入工具目录页
3. 筛选"免费" → 看到所有免费工具
4. 按评分排序 → 看评分高的工具
5. 点击工具卡片 → 进入详情页
6. 看完介绍觉得合适 → 点击"访问官网" → 开始使用
```

### 流程二：AI推荐找工具
```
1. 用户打开网站 → 在首页输入框说需求
2. 比如输入："我是学生，需要写论文降重，有什么免费AI工具推荐？"
3. 点击发送 → 等待AI思考几秒
4. AI给出3个推荐结果，分别说明优缺点
5. 用户看推荐理由，选一个最合适的 → 点击进详情页
6. 去官网试用
```

### 流程三：看榜单选工具
```
1. 用户点击导航栏"榜单" → 进入榜单列表
2. 看到"2025年最好用的AI写作工具TOP5" → 点击进入
3. 从上往下看每个工具的点评 → 第一名正好符合需求
4. 点击工具卡片 → 详情页 → 访问官网
```

### 流程四：按场景找方案
```
1. 用户点击导航栏"场景" → 进入场景列表
2. 找到"电商主图设计"场景 → 点击进入
3. 看完场景介绍，看推荐工具：入门用A，专业用B，免费用C
4. 自己是新手，选免费的C → 点击去详情 → 使用
```
