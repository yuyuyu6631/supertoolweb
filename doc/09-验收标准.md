# AI 工具评测平台验收标准

## 一、总体要求

- 面向中文用户，主要文案为中文
- 页面需要保持当前玻璃工作台风格，不退回旧版 Demo 视觉
- 当前验收以真实代码现状为准，不以历史设计稿或旧技术栈为准
- 前端、后端、认证、测试与文档口径必须互相一致

## 二、前端验收标准

| 编号 | 项目 | 验收说明 |
| --- | --- | --- |
| F-1 | 项目可启动 | 根目录执行 `python start.py` 或 `npm run dev` 后，首页可访问 |
| F-2 | 页面路由 | `/`、`/tools`、`/tools/[slug]`、`/rankings`、`/scenarios`、`/scenarios/[slug]`、`/auth` 可访问 |
| F-3 | 目录数据读取 | 首页与工具目录从真实 API 读取数据，加载失败时有回退或错误提示 |
| F-4 | 目录筛选 | 搜索、分类、标签、状态、排序、分页可工作 |
| F-5 | 登录态展示 | Header 能正确显示“登录 / 注册”、用户名、退出按钮和登录态加载状态 |
| F-6 | 认证页面 | `/auth` 提供登录、注册切换，表单校验、提交中状态和错误反馈正确 |
| F-7 | 响应式 | 主要页面在桌面和移动端布局正常，无明显溢出 |
| F-8 | 路由反馈 | 页面切换时有 `RouteFeedback` 反馈，不影响正常导航 |
| F-9 | 不报错 | 浏览器控制台不应出现明显的 React 运行时错误 |

## 三、后端验收标准

| 编号 | 项目 | 验收说明 |
| --- | --- | --- |
| B-1 | 目录接口 | `GET /api/tools`、`GET /api/tools/{slug}`、`GET /api/rankings`、`GET /api/scenarios` 正常返回 |
| B-2 | 推荐接口 | `POST /api/recommend` 返回结构稳定；`GET /api/ai-search` 是 AI 搜索入口（与 recommend 是两个独立端点） |
| B-3 | AI 搜索接口 | `GET /api/ai-search` 返回 AiSearchResponse（含 ai_panel、results、directory），AI Panel 和快捷动作正常展示 |
| B-3 | 认证接口 | 注册、登录、登出、读取当前用户接口可用 |
| B-4 | 会话能力 | 登录后可通过 Cookie 恢复登录态，登出后会话失效 |
| B-5 | 错误处理 | 认证失败、资源不存在、参数错误时返回清晰错误结构 |
| B-6 | 抓取入口 | `POST /api/crawl/jobs` 可调用，当前返回轻量抓取结果 |
| B-7 | 数据模型 | Alembic 迁移和 SQLAlchemy 模型覆盖目录、抓取和认证实体 |

## 四、测试验收标准

| 编号 | 项目 | 验收说明 |
| --- | --- | --- |
| T-1 | Web 单测 | `npm run test:web` 可执行，核心页面和关键组件测试存在 |
| T-2 | Auth API 测试 | `apps/api/tests/test_auth_api.py` 存在并覆盖认证主流程 |
| T-3 | E2E | Playwright 用例覆盖 smoke、visual、auth checklist |
| T-4 | 文档一致性 | `doc/README.md`、`08`、`10`、`12`、`13`、`14` 的技术栈和路由描述一致 |

## 五、当前明确不作为验收前提的内容

以下内容不应再作为当前版本的必验项：

- 基于 `Vite` 或 `React Router` 的旧路由实现
- 浏览器侧 `mock API` 拦截
- `/rank`、`/categories/[slug]` 等旧页面路径
- “登录注册不做”的旧边界
- “所有页面只靠 JSON/mock 数据渲染”的旧前提

## 六、验收方式

1. 通过代码检查确认真实路由、接口和模型与文档一致
2. 运行前端测试与后端测试，确认核心链路不退化
3. 在 `doc/` 下扫描旧关键词，确保历史技术栈没有再被写成当前现状
