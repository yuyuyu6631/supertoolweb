# 06 数据结构

## 1. 当前实现说明

当前项目已经使用真实数据库模型，不再以“仅靠静态 mock 数据即可”作为系统基线。  
前端仍保留 `src/data/tools.json` 和 `src/data/scenarios.json` 作为仓库内数据资产，但当前核心页面读取真实 API，后端以 MySQL + SQLAlchemy 模型为准。

## 5. 当前数据来源边界

- 前端页面展示的目录、榜单、场景、工具详情以 API 为主
- 后端 schema 以 `apps/api/app/schemas/*.py` 为准
- 前端本地类型定义位于 `apps/web/src/app/lib/catalog-types.ts` 与 `apps/web/src/app/lib/auth-api.ts`
- `packages/contracts` 仍保留共享类型和样例数据，但前端当前没有完全切换到共享包直引

## 6. 后续演进说明

未来若继续推进：

- 可以把前端本地类型进一步收敛到 `packages/contracts`
- 可以把抓取快照和审核发布链路从“有表有入口”升级为完整工作流
- 可以在不改变公开路由的前提下继续扩展认证与后台模型
