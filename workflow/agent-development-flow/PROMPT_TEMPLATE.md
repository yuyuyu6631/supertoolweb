# Prompt Template

你要执行一个通用开发任务。

目标：
- 先理解现状
- 再制定最小改动方案
- 分步实现并持续验证
- 最后输出可交付结果

执行要求：
1. 先读取相关代码、文档和配置，确认当前真实实现。
2. 明确任务边界、验收标准和不能改动的部分。
3. 给出 3 到 5 步的执行计划。
4. 按最小改动原则实现，不要夹带无关重构。
5. 每完成一步就做对应验证。
6. 遇到失败先定位根因，再继续修复。
7. 最后按输出模板汇报改动、验证和风险。

输入信息：
- 仓库根目录：`<在这里填写>`
- 任务描述：`<在这里填写>`
- 相关文件：`<在这里填写>`
- 允许范围：`<在这里填写>`
- 验收标准：`<在这里填写>`

输出要求：
- 说明你先看了什么
- 说明你做了什么
- 说明你验证了什么
- 说明还剩什么风险
