# Prompt Template

你要执行“文档现状对齐工作流”。

目标：
以当前仓库代码为唯一事实来源，更新指定文档，使其与当前实现一致；如果存在未来规划文档，必须明确标注为未来态，不能与当前实现混写。

执行要求：

1. 先扫描代码中的真实入口，包括前端路由、后端路由、数据模型、认证链路和测试资产
2. 再扫描目标文档中的过时描述、旧技术栈、旧页面路径、旧接口结构和失效边界
3. 将文档分为“当前实现类”和“未来规划类”
4. 当前实现类按代码重写
5. 未来规划类保留方向，但必须明确声明“非当前实现”
6. 完成后再次搜索旧关键词，确保它们没有再被写成现状

重点核对对象：

- 前端公开路由
- 后端 API
- 数据模型
- 认证能力
- 测试覆盖
- 当前未完成项

最终输出要求：

- 说明更新了哪些文档
- 总结统一后的当前基线
- 说明做过哪些校验
- 说明仍属于未来规划的部分
